# pickpix图片挑选工具

## 简介

Pickpix 是一个基于 Python + Tkinter 的多方法图片对比裁剪工具，适合以下场景：

- 多个方法结果并排对比查看
- 在同一坐标下对所有方法统一裁剪
- 支持本地目录与 SFTP 远程目录输入/输出
- 支持批量处理全部帧

当前版本的输入扫描规则为：

- 仅扫描文件名匹配 frame数字.exr 或 frame数字.png 的序列
- 例如 frame0001.exr、frame12.png

说明：程序内部会优先尝试读取 EXR，其次 PNG。输出统一保存为 PNG。

## 环境配置

依赖见 requirements.txt：

- Pillow
- numpy
- opencv-python
- paramiko
- PyYAML

pip install -r requirements.txt

## 代码运行

``` bash
python pickpix.py
```

## 输入数据组织

支持两种本地输入方式。

方式 A：选择一个总目录，目录下每个子目录视为一个方法。

```text
input_root/
  method_a/
    frame0001.exr
    frame0002.exr
  method_b/
    frame0001.png
    frame0002.png
```

方式 B：直接选择某个方法目录（目录本身包含 frame 序列）。

```text
method_a/
  frame1.exr
  frame2.exr
```

远程输入与本地规则一致，只是路径在服务器上。

## 快速上手（推荐流程）

1. 点击 添加输入文件夹（或 添加远程输入）。
2. 点击 选择输出文件夹（或 选择远程输出）。
3. 如需保留当前工作状态，可点击 保存工程 / 工程另存。
3. 在左侧预览区切换到目标帧。
4. 鼠标左键拖拽绘制裁剪框。
5. 点击 添加到裁剪列表。
6. 可重复添加多个裁剪框。
7. 执行 批量裁剪当前帧 或 批量裁剪所有帧。

## 交互说明

### 方法列表

- 方法列表支持勾选显示、单独设置每个方法的帧偏移
- 点击某一行的 克隆，可复制一个同源方法，方便设置不同偏移对比同一组图
- 点击某一行的 移除，可把该方法从当前列表移除；如果有依赖它的差分方法，会一并移除
- 点击 生成差分，可选择两个方法，生成一个新的 errormap 方法

说明：

- 克隆方法与差分方法仅保留在当前会话中，重新扫描或重启后不会持久化
- errormap 方法按两个方法当前帧结果的绝对像素差生成

### 工程文件

- 支持 保存工程、工程另存、导入工程
- 工程文件会保存输入源、输出目标、当前帧号、方法显示状态、偏移、派生方法、裁剪框、书签和预览大小
- 关闭软件、导入其他工程、清空输入前，会提示是否先保存当前工程
- 远程 SFTP 凭据会一并写入工程文件，便于恢复，但工程文件中会包含明文密码

### 书签

- 可收藏 / 取消收藏当前帧
- 支持上一书签、下一书签
- 可从书签下拉列表直接跳转
- 书签会随工程文件一起保存和恢复

### 帧与视图

- 上一帧 / 下一帧：循环切换帧
- 跳转：输入帧号后跳转
- 鼠标滚轮：滚动当前区域
- Ctrl + 鼠标滚轮：缩放（0.1x 到 5.0x）
- 重置1:1：恢复原始缩放并居中
- 右键拖拽：平移查看大图

注意：切换帧时会清空当前已添加的裁剪框列表。

### 裁剪框

- 左键拖拽：绘制矩形框
- 按住 Shift + 左键拖拽：绘制正方形框
- 添加到裁剪列表：保存当前框
- 裁剪框列表双击：删除对应框
- 清空所有裁剪框：删除全部框

### 手动坐标

右侧可手动输入：

- X、Y
- 宽度、高度

点击 应用坐标 后会立即更新预览和各方法画布上的框。

## 输出内容说明

### 批量裁剪当前帧

会输出：

- 每个方法下的裁剪图：frame帧号_box1.png、frame帧号_box2.png ...
- 每个方法下的可视化框图：frame帧号_boxes_map.png
- 输出根目录下的当前帧总览图：frame帧号_summary.png

这里的方法包括：

- 扫描得到的原始方法
- 克隆出来的方法
- 生成的 errormap 方法

### 批量裁剪所有帧

会输出：

- 每个方法每一帧的裁剪图
- 每个方法每一帧的可视化框图

说明：差分方法会在每一帧实时生成绝对像素差图后参与裁剪输出。

说明：该模式不会生成 summary 拼图。

### 输出目录示例

```text
output_root/
  method_a/
    frame0001_box1.png
    frame0001_box2.png
    frame0001_boxes_map.png
  method_b/
    frame0001_box1.png
    frame0001_box2.png
    frame0001_boxes_map.png
  frame0001_summary.png
```

## 远程 SFTP 配置

服务器预设来自 config/paths.yaml 的 servers 节点。

示例结构：

```yaml
servers:
  server_1:
    label: Server 191
    host: 1.2.3.4
    port: 22
    username: your_user
    password: your_password
```

你可以新增多个服务器预设，界面会自动读取并显示。

远程路径要求：

- 必须是绝对路径
- 必须以 / 开头
